
[使用平台]
Windows 2000/Windows XP

備註：Windows 2000 目前看起來無法正確支援 Unicode 3.1 B 的中文字，所以如果遇到
這類的字處理會不正確。目前會試著檢查是否有 Unicode ExtB 的字型，如果沒有則使用
顯示空白的方式來表示。

[安裝方式]

執行本程式除主程式、MinGW runtime 和 Keyboard hook DLL 外，尚需要下列的輸入法參
考檔：

	主檔案：array30.cin
	簡碼：ArrayShortCode.cin
	詞庫：arbox.txt （for 日文假名輸入）

以及下列二個設定檔案：

	listclass.txt
	listtitle.txt


[使用說明]

快速鍵

	Shift 切換中/英文輸入
	CTRL + SPACE 切換行列/英數模式
	SHIFT + SPACE 切換全形/半形模式

支援 United State - Dvorak keyboard layout

	目前做法：檢查目前的 keyboard layout，如果是 0xF0002，就判定
	為 Dvorak 鍵盤，就對應到 QWERTY 鍵盤的相對應位置 key code。

	好處不論是 QWERTY/Dvorak 都使用相同的表格，缺點則是當使用者使
	用 Dvorak 鍵盤時，少部份符號鍵會被對應為 QWERTY 鍵盤的 KEY。

	這樣行列輸入法的符號鍵和詞鍵位置會完全相同，亦即在大多數的情況
	不會因為 QWERTY/Dvorak 鍵盤的不同而使得按鍵位置有所改變。

送字方式

	提供了下列的送字方式：
	使用剪貼薄送字（SendInput 送出 SHIFT+INSERT / CTRL + V）
	使用 WM_IME_CHAR 訊息
	使用插入方法到微軟提供的 Edit/RichEdit Control 元件（適用於記事本、Wordpad）

	程式有自動切換送字功能的選項，啟動後如果遇到 Edit/RichEdit Control 元件，會
	自動切換到該模式送字，這樣可以增加執行時的效率。

	在開啟自動切換送字功能後，除了使用插入方法到微軟提供的 Edit/RichEdit Control 元
	件這個方式外，使用者可以在 listclass.txt/listtitle.txt 自行設定當開啟自動切換送
	字功能時要切換的送字方式。

	設定：
		[Windows Class],[送字方式]
		[Windows Title],[送字方式]

	送字方式：
		1. 使用 WM_IME_CHAR 訊息
		2. 使用剪貼薄 (SHIFT+INSERT) 送字
		3. 使用剪貼薄 (CTRL + V) 送字
		4. 使用插入方法到微軟提供的 Edit/RichEdit Control 元件
		5. 使用 GTK+ 所撰寫的程式

	在設定後，重開 Array30Ex 設定值即會生效。

符號輸入處理

	當按 [W]鍵，再按某個數字鍵時，就會列出符號群組供使用者選擇。
	
	[W]鍵 + 數字鍵[1] →  標點符號組
	[W]鍵 + 數字鍵[2] →  括號符號組
	[W]鍵 + 數字鍵[3] →  一般符號組
	[W]鍵 + 數字鍵[4] →  數學符號組
	[W]鍵 + 數字鍵[5] →  方向符號組
	[W]鍵 + 數字鍵[6] →  單位符號組
	[W]鍵 + 數字鍵[7] →  圖表符號組
	[W]鍵 + 數字鍵[8] →  順序符號組
	[W]鍵 + 數字鍵[9] →  希臘符號組
	[W]鍵 + 數字鍵[0] →  注音符號組

萬用字元 * 與 ?

	為了讓使用者在部份不知道的字根鍵位可以用 Wildcard 的觀念來處理，這裡提供
	了「?」及「*」來代替。
	
	查詢的情況下，經常會有一頁顯示不完的情形，翻頁的處理跟符號輸入的翻頁處理
	一樣，仍用空白鍵或[Shift]+[<]及[Shift]+[>]來處理。
	
詞庫與日文假名

	行列的詞庫按「’」或者是「”」鍵來進行輸入。
	
	這個程式使用詞庫的方式支援日文假名的輸入，但是並沒有對詞庫進行嚴謹的處理，僅
	為了支援日文假名輸入的用途。		
	
特殊編碼

	「，，ＯＰＴ」 開啟設定對話框
	「，，ＰＬＣ」 輸入法列回到預設位置
	「，，ＯＵＴ」 離開

繁簡即時轉換

	這裡的實作是參考自香草輸入法，以表格對照的方式來實作繁簡即時轉換，
	實作打繁體輸出簡體，或者是打簡體輸出繁體的功能

設定視窗透明度
	右鍵選單實作「設定視窗透明度」選項，可以在程式執行期間動態改變視窗透明度。
